QPM Rolling Backtest — Phase 2

Pseudo-real-time forecasts using the JS QPM solver in qpm_uzbekistan/index.html, scored against actual Uzbek macro outcomes 2018Q1–2025Q3. See validation-note.md §4 for interpretation.
Methodology caveat. Historical state at each forecast origin comes from the MATLAB IRIS Kalman-smoothed decomposition (data/matlab_kalman_filter.csv). This uses information from the full sample to estimate trends, so the backtest is not fully real-time — it isolates the QPM's gap-forecasting dynamics from the separate problem of trend estimation. Phase 3 (re-filter at each origin) would give true real-time results.

Methodology — full detail

  1. Forecast origins. Each quarter t₀ from the start to the end date above.
  2. Initial deviation state at t₀, read from the Kalman-smoothed series:
    • gap₀ = L_GDP_GAP(t₀)
    • π₀ = D4L_CPI(t₀) − D4L_CPI_TAR(t₀) (inflation deviation from target)
    • rs₀ = RS(t₀) − RSNEUTRAL(t₀) (policy rate deviation from neutral)
    • d4ls₀ = 0 (NER YoY deviation from steady-state path; held at 0 for simplicity)
  3. Forecast. Call solveIRF(p, null, 0, h, initConds) — no shock, deterministic forward projection in deviation form, with the JS solver's 5-quarter pre-history filled at the initial state.
  4. Level reconstruction using the smoothed trends at the forecast date:
    • D4L_CPI_pred(h) = D4L_CPI_TAR(t₀+h) + π_pred(h)
    • RS_pred(h) = RSNEUTRAL(t₀+h) + rs_pred(h)
    • D4L_GDP_pred(h) ≈ DLA_GDP_BAR(t₀+h) + gap_pred(h) − gap(t₀+h−4) (YoY growth = trend + change in gap)
  5. Score against actuals (OBS_*) at t₀+h:
    • RMSE = √(Σ(pred − actual)²/N) — penalises large errors quadratically
    • MAE = Σ|pred − actual|/N — average absolute error
    • Directional accuracy = % of origins where sign(pred(h) − actual(t₀)) matches sign(actual(t₀+h) − actual(t₀))
