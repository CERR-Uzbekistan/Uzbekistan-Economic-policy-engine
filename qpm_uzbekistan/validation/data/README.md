# QPM Validation Data — Provenance

Source pack: **CERR Models / QPM_UZB** (MATLAB / IRIS Toolbox reference implementation).
Imported into the repo: **2026-05-25** for issue #141 (QPM v1 validation pack).

All files in this folder are read-only artefacts used to validate the
HTML QPM in `qpm_uzbekistan/index.html` against the canonical MATLAB
implementation. They are **inputs to the validation pack**, not
auto-generated by the web platform — do not regenerate from JS.

## Transformations applied by MATLAB pack (replicated for validation)

1. **Monthly → quarterly**: arithmetic mean of three monthly observations within each quarter (`a02_makedata_uzb.m:18-21`).
2. **Seasonal adjustment**: 4-quarter trailing moving average on raw CPI and GDP to remove the "sawtooth" pattern in Uzbek seasonal data (`a02_makedata_uzb.m:38-49`).
3. **Log-transformation**: `L_GDP = 100·log(GDP_SA)`, `L_CPI = 100·log(CPI_SA)`, `L_S = 100·log(S)` (`a02_makedata_uzb.m:57-62`).
4. **OBS_ mapping**: identity mapping from transformed series to model observables — `OBS_L_GDP = L_GDP`, `OBS_L_CPI = L_CPI`, `OBS_RS = RS`, `OBS_L_S = L_S` (`a02_makedata_uzb.m:66-69`).
5. **Foreign-block stubs** (used in Kalman filter only): foreign output gap = 0, foreign inflation = 4 % p.a., foreign rate = 2 % p.a., domestic target = 5 % p.a. (`a02_makedata_uzb.m:73-76`). These are placeholders that should be replaced with WEO / Fed series in Phase 2.

## Missing-data handling

- `ulc_u`: `NaN` for 2016 (series begins 2017Q1). Treat as missing.
- `intr_u`: zeros for 2016–2017 (series begins 2018Q1).
- `cpi_*_u` foreign CPI for 2016M01–2016M03: identical placeholder values (100 / 99.6 / 99.4 / …). Filtered out for the foreign block.
- `ir_lend_*_lc/usd`: `NaN` for 2016M01–2016M12 (lending-rate series begins 2017M01).

The MATLAB pack uses `fillmissing(d.X, 'linear')` on the 4-quarter moving averages (`a02_makedata_uzb.m:52-53`). Any JS-side backtest must apply the same imputation before computing transformations.

## Do not edit

- The CSVs are **byte-identical copies** of the MATLAB pack output. Do not reformat (changing the delimiter or decimal separator will break consumers).
- `Uzbekistan.model` and `readmodel_uzb.m` are kept here purely as the reference equation specification. They are not executed by the web platform.
- Regenerate by re-running the MATLAB pipeline (`a02 → a03 → a07`) and copying the outputs back; document the new vintage on this README.
