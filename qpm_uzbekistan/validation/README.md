## How to run the cross-check locally

The page uses `fetch()` to load CSVs, so it needs to be served over HTTP (not opened as `file://`).

```bash
# from the repo root
python -m http.server 8123
# then open: http://localhost:8123/qpm_uzbekistan/validation/cross-check.html
```

In Claude Code, the launch config at `.claude/launch.json` is named **`qpm`** — `preview_start name=qpm` brings up the same server.

## Maintenance

- The CSVs under `data/` are **byte-identical copies** of the CERR MATLAB pack output. To refresh: re-run `a02 → a03 → a07` in the MATLAB pack, copy outputs back, bump the vintage in [data/README.md](data/README.md).
- The QPM solver in [cross-check.html](cross-check.html) is a **verbatim copy** of `qpm_uzbekistan/index.html:571–724`. If the main solver changes, refresh the copy here and re-run the cross-check.
- This folder must not contain any data that is generated by the JS engine — only inputs and reference artefacts.
